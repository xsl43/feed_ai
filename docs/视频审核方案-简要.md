# 视频审核功能设计方案

## 一、概述

基于 AI 大模型（LLM + Vision Model）的自动内容审核系统。**先审后发**——视频提交后默认不可见，审核通过才进入 Feed。

**核心原则：**

- **分层审核**：基础校验 → AI 多维度审核 → 人工复核
- **多维度并发**：文本、封面、帧、音频、OCR 五路并行，帧内再并发
- **灰区转人工**：AI 不确定的不粗暴打回，进入人工队列
- **流量越热越严**：热门/被举报内容触发事后复审

---

## 二、审核链路

### 2.1 数据流

    上传 → 格式校验 → 发布 → 敏感词预检 → 抽帧 → 5维并发AI审核 → 决策矩阵 → 通过/打回/转人工
      │       │                                      │
      │       └─ 魔数 + ffprobe + 大小               └─ 全部维度结果取最差
      └─ 前端 accept 扩展名 → 后端二次校验

### 2.2 基础校验层（不调 LLM，零成本）

| 校验项 | 方法 | 作用 |
|--------|------|------|
| 扩展名 | 白名单 .mp4/.mov/.avi/.mkv/.webm | 前端 + 后端双重校验 |
| 魔数 | 读文件头 12-16 字节比对 | 防格式伪装（exe 改名 mp4） |
| 流完整性 | ffprobe 解析 | 防内容损坏（传一半断网） |
| 文件大小 | 视频 ≤500MB，封面 ≤20MB | 可配置 |

### 2.3 敏感词预检（DFA）

启动时加载 configs/sensitive_words.txt，用 DFA 状态机 O(n) 扫描标题和描述。命中 → 直接打入 manual_review，跳过 LLM，省费用。

### 2.4 AI 审核管道（5 维并发）

    Publish() → go ReviewAndPublishVideo(video)
      ├─ goroutine 1: 文本审核   DeepSeek-V3        审标题+描述
      ├─ goroutine 2: 封面审核   Qwen3-VL           审封面图
      ├─ goroutine 3: 帧审核     Qwen3-VL ×3帧      抽帧并发审
      ├─ goroutine 4: 音频审核   ASR → DeepSeek-V3  转录文本审核
      └─ goroutine 5: OCR审核    PaddleOCR → DeepSeek-V3  帧内文字审核

**帧提取**：ffmpeg 按时间轴均匀采样 3 帧（可配置），以 data URL 传给 Vision Model。

**并发控制**：帧内 3 帧用 semaphore 限制并发数；全局 worker pool 限制最多 10 个视频同时在审。

### 2.5 决策矩阵

LLM 返回 {"status":"approved/rejected","confidence":0.0~1.0}：

| AI判定 | conf ≥ 0.70 | 0.50 ≤ conf < 0.70 | conf < 0.50 |
|--------|-------------|---------------------|-------------|
| approved | **通过** | 转人工 | 转人工 |
| rejected | **打回** | 转人工 | **打回**（安全兜底） |

多维度结果取最差：任一维度 rejected 则整体 rejected。低置信度 rejected 仍然打回（宁可误杀不漏放）。

### 2.6 事后复审

| 频率 | 触发条件 | 动作 |
|------|---------|------|
| 每 5 分钟 | popularity > 10000 | 重新文本审核 |
| 每 5 分钟 | report_count ≥ 10 | 重新文本审核 |
| 每 1 小时 | play_count > 50000 | 重新文本审核 |

## 三、模型配置

全部通过 SiliconFlow API（OpenAI 兼容），一个 API Key 通吃：

| 用途 | 模型 | 配置项 |
|------|------|--------|
| 文本审核 | deepseek-ai/DeepSeek-V3 | review.text_model |
| 图片审核 | Qwen/Qwen3-VL-32B-Instruct | review.vision_model |
| OCR文字提取 | PaddlePaddle/PaddleOCR-VL-1.5 | 代码内硬编码 |
| 语音转文字 | TeleAI/TeleSpeechASR | ai.asr_model |

调用方式：POST https://api.siliconflow.cn/v1/chat/completions，统一在 review/service.go 中封装 callTextLLM / callVisionLLM / callOCR。

---

## 四、审核状态机

    pending → AI审核 → approved  → Feed可见
                      → rejected  → 通知用户
                      → manual_review → 管理员审核 → approved/rejected

事后复审：approved → 触发规则 → 重新审核 → 可能变为 rejected。

---

## 五、人工审核队列

### 优先级计算

| 维度 | 权重 |
|------|------|
| 涉政 | +100 |
| 暴恐 | +80 |
| 色情 | +60 |
| 辱骂 | +40 |
| 广告 | +20 |
| 多维度命中 | 每多一维 +30 |
| 帧审核命中色情/暴恐 | +50/+80 |
| 被举报 > 5/20 次 | +50/+100 |

置信度越低越优先。队列按 review_priority DESC, create_time ASC 排序。

## 六、API 接口

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| GET | /review/config | JWT | 获取审核配置 |
| POST | /review/config | Admin | 更新审核配置 |
| GET | /review/status/:id | JWT | 查询视频审核状态 |
| POST | /review/resubmit | JWT | 修改后重新提交 |
| GET | /review/pending | Admin | 待审列表（按优先级） |
| POST | /review/video/:id/approve | Admin | 人工通过 |
| POST | /review/video/:id/reject | Admin | 人工拒绝 |
| POST | /video/delete | JWT | 删除视频 |

## 七、文件结构

    backend/
    ├── configs/
    │   ├── config.yaml              # 审核配置
    │   └── sensitive_words.txt      # 敏感词库
    ├── internal/
    │   ├── review/
    │   │   ├── entity.go            # ReviewConfig, ReviewResult
    │   │   ├── sensitive.go         # DFA 敏感词过滤器
    │   │   └── service.go           # LLM调用、决策矩阵、多维度聚合
    │   ├── video/
    │   │   ├── video_entity.go      # Video 实体（含审核字段）
    │   │   ├── video_handler.go     # 上传：魔数+ffprobe+大小校验
    │   │   └── video_service.go     # 发布入口 + 审核触发
    │   └── worker/
    │       └── reviewworker.go      # 事后复审定时任务
